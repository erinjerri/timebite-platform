# Pricing (starter hypothesis)

This doc is a working hypothesis. Pricing should change as we learn what users truly value.

## Positioning anchor
TimeBite is a **focus operating system** with execution truth. It competes with:
- lightweight focus apps (timers)
- task planners (but we are not trying to be “another Notion”)
- habit systems (but we’re time-budget + execution evidence)

## Core value users pay for
- reliably protecting deep work time
- seeing the truth of where time went
- reducing context switching and reactive drift

## Proposed tiers (initial)

### Free
- Daily Intent planning (limited history)
- Ring timer + session logs
- Dashboard Today view

### Pro (subscription)
- Week/month history + trends
- Advanced dashboards and export
- Capture features (voice + vision) if they incur cost
- Optional AI assistance features
- (Later) multi-device sync

## Pricing hypothesis
- $7–$12 / month for Pro, depending on feature set and ongoing costs.

## Experiments (Q2 2026)
- Measure willingness-to-pay via:
  - waitlist survey
  - in-app “unlock Pro” taps
  - pricing page conversion (when public)
- Identify which features drive upgrade intent:
  - history + trends
  - capture workflows
  - AI assistance

## Risks
- Overpricing before product-market resonance.
- Charging for core loop too early (keep core habit formation accessible).
