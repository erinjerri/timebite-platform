# Dashboard Specification

The dashboard is a summary surface, not the MVP's main work surface.

---

## Product Role

Dashboard exists to reinforce the loop:

```text
Goal Loop -> Execution -> Reflection -> Streak -> Insights
```

It should help users feel continuity, not pressure.

---

## MVP Sections

1. Current streaks
2. Goals practiced this week
3. Monthly completion calendar preview
4. Weekly summary
5. Monthly summary
6. Recent reflections
7. Light insights

---

## Deferred Sections

- Dense analytics charts
- Time allocation bars
- Cycle score
- Quarterly budget chart
- AI-generated coaching
- Comparative productivity scoring

---

## Empty States

Before enough data exists, dashboard should send users back into the loop:

- Set today's intent.
- Start a small session.
- Complete one goal.
- Write one sentence.

---

## Insight Rules

Insights should be:

- derived from completed sessions and reflections
- short
- explainable
- non-judgmental
- secondary to action

Example: "You completed creative work three mornings this week. Morning may be your easiest time to keep this goal small and repeatable."
