# Launch Checklist (iPhone MVP)

This is a founder-friendly checklist to ship the iPhone MVP without overbuilding.

## Product
- [ ] Daily Intent planning (commit flow)
- [ ] Focus lanes (3–4 max; one primary task per lane)
- [ ] Ring timer execution (durable session logs)
- [ ] Reflection (daily + session note)
- [ ] Dashboard Today view (truth from session logs)

## Quality
- [ ] Crash-free sanity pass (TestFlight)
- [ ] Offline mode verified (airplane mode)
- [ ] Background timer behavior verified
- [ ] Midnight/time zone rules documented
- [ ] Empty states for first-run and “no sessions yet”
- [ ] Accessibility basics (Dynamic Type, VoiceOver labels)

## Analytics (privacy-first)
- [ ] Event allowlist defined (`analytics/README.md`)
- [ ] Core funnel tracked:
  - open app → commit intent → start timer → end session → write reflection

## Security + privacy
- [ ] No secrets committed
- [ ] Privacy policy published
- [ ] Permissions explained in onboarding/help (microphone/camera if used)

## App Store
- [ ] App Store Connect record created
- [ ] Screenshots captured (real UI)
- [ ] Description written (clear value prop)
- [ ] Review notes prepared
- [ ] Versioning + release notes ready

## Operational
- [ ] Simple support channel (email)
- [ ] Lightweight feedback collection (in-app link)
- [ ] Release plan for first 2 updates (bugfix cadence)
