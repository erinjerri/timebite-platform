# Backlog (prioritized)

This backlog is intentionally **lean**. It is not meant to be exhaustive; it is a decision tool.

## P0 — Core loop hardening (iPhone)
- Onboarding that creates first day plan
- Undo for destructive actions (delete task, end timer)
- Session overlap + midnight attribution rules
- Accessibility pass (Dynamic Type + VoiceOver)
- Export: JSON export of day plan + sessions (debug/user ownership)

## P1 — Capture + workflow acceleration
- Voice capture → propose tasks → confirm
- Vision/OCR capture → propose tasks → confirm
- Quick add from lock screen / widgets (later iOS)
- “Plan suggestions” based on history (AI optional)

## P1 — Constraints and realism
- Project buckets + weekly hour caps (Time Allocation Engine)
- Project states: active/paused/deferred/blocked/archived
- Energy-aware task labeling + “suggested next” recommendations

## P1 — Dashboard expansion
- Week review: lane trends, streaks, variance
- “Intent accuracy” visualization (target vs actual)
- “Most effective hour” / “context switch” heuristics (careful with accuracy)

## P2 — Integrations (later)
- Calendar read-only import (events → time blocks)
- Notion import/export (tasks)
- Sunsama adapter (if strongly demanded)

## P2 — Optional backend sync (later)
- Auth + device identity
- Sync service + conflict resolution
- Encryption strategy for sensitive fields

## P3 — Platform expansion (later)
- watchOS “Now” companion (start/pause/complete)
- macOS planning/review surface
- visionOS ambient rings (research)
