# Specs

This folder is the requirements-first source of truth for TimeBite’s product direction and delivery cadence.

Conventions:
- Write in terms of user outcomes and measurable acceptance criteria.
- Prefer iPhone-first scope. Treat watchOS/macOS/visionOS as follow-ons unless explicitly scheduled.
- Assume local-first persistence now; treat backend sync as optional, later.

Key files:
- `vision.md` — long-term product vision and principles
- `master-requirements.md` — core feature requirements (functional + nonfunctional)
- `user-stories.md` — story catalog for planning and testing
- `roadmap-q2.md` — Q2 (Apr–Jun) delivery plan
- `sprint-current.md` — current sprint objectives and tasks
- `backlog.md` — prioritized backlog
