# TimeBite Spatial Environment Engine

The Spatial Environment Engine is a semantic styling layer for adaptive dashboard interfaces. It is inspired by LunarVim profiles, terminal themes, and CSS variables, but it is designed for TimeBite's spatial product surface instead of text editors or web pages.

The flow is intentionally small:

```text
Prompt or config
-> SpatialEnvironment
-> DesignTokens
-> SwiftUI and RealityKit rendering
-> TimeBite ritual experience
```

This is not a theme switcher. A theme switcher changes colors. A spatial environment changes the interface posture: how quiet Today feels, how present the timer is, how softly the monthly calendar reinforces completion, how much motion is allowed, and how each surface expresses the same environment.

## Semantic Model

`SpatialEnvironment` describes user intent before it describes pixels.

- `intent`: focus, operator, or flow
- `mood`: calm, energetic, ambient, or analytical
- `style`: minimal, glass, terminal, softNeo, or animeNeon
- `energy`: low, medium, or high
- `density`: minimal, balanced, or dense
- `brightness`: light, dark, or dim
- `preferredSurfaces`: per-tab emphasis and density

`TokenResolver` turns those semantics into `DesignTokens`. Views consume tokens only. They should not hardcode background colors, card colors, spacing, typography, animation timing, or ring colors.

## TimeBite Surfaces

TimeBite has four primary MVP surfaces:

- Today: daily intent, active session, completion checkmarks, and reflection prompt
- Goals: goal setup, frequency, streak state, tags, and recent sessions
- Calendar: monthly completion state, day detail, sessions, and reflections
- Dashboard: current streaks, weekly summary, monthly summary, recent reflections, and light insights

Each environment may shape these surfaces differently. Focus can make Today minimal. Operator can make Dashboard slightly denser while keeping analytics secondary. Flow can soften reflection and ambient panels while preserving the core TimeBite navigation.

## Constraints

Environment configs should preserve:

- Core tab navigation
- Torus timer geometry
- Accessible text contrast
- Safe motion intensity
- Predictable card and panel hierarchy
- TimeBite's restrained product identity

User expression should be constrained, not chaotic. The engine should let people tune the feeling of their workspace without breaking the dashboard's usability.

## AI-Safe Spatial Design Principles

Spatial configuration can be generated by AI, but the renderer should remain opinionated.

- Treat intent as semantic input, not visual decoration.
- Resolve ambiguous prompts into safe defaults.
- Cap motion at safe levels.
- Keep brightness and contrast within readable ranges.
- Keep per-surface density bounded.
- Preserve navigation and primary workflows.
- Prefer quiet adaptation over novelty.

The goal is an expressive ritual surface that still feels like TimeBite: focused, readable, and calm enough to use every day.
